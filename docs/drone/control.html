1. Протокол взаимодействия

Является собственной технологий взаимодействия. Дрон считывает каждый новый пришедший байт и записывает его в буффер, размер которого равен размеру пакета - 7 байтам, до того момента пока пакет не будет распознан. Каждый пакет имеет строгую структуру, как пакет установки значений, так и пакет получения значений. Первый байт пакета отвечает за тип сообщения - установка (1) или получение (0) значения.

Второй байт пакета отвечает за номер параметра, который требуется получить или установить. Следующие 4 байта отводятся для хранения значения для установки ( в случае получения значений эти 4 байта используюся только для вычисления корректности пакета, о чем будет рассказано ниже).

Последний завершающий байт используется для подтверждения корректности пакета ( проверяет был ли пакет поврежден при отправке или является ли это пакетом, а не шумами при передаче информации) и является сложением по модулю два всех предыдущих байтов.

Рекомендация

При отправке пакета с получения ифнормации, желательно заполнять 4 байта значений рандомными байтами для обеспечения откаустойчивости передачи данных.

2. Подключение к дрону

Как управление, так и подключение к дрону осуществляется с помощью специльного программного обеспечения. Следуйте указаниям и интуитивно понятному интерфейсу программы. Скриншоты программы:

Для входа в программу необходимо ввести логин и пароль. Приложение делится на несколько тематических разделов. На главном экране отображена информация о профиле. В разделе "Дрон" присутствует весь необходимый функционал для работы с дроном. Раздел заказы предназначен для управления заказами компании. Для обеспечении технической поддержки предназначен раздел "Чат".
